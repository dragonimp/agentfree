# AgentFree 聊天功能测试案例

## 一、会话列表功能

### TC-01：会话列表按智能体名称分组显示

**前置条件：** 已创建至少2个不同智能体的会话

**测试步骤：**
1. 打开聊天页面（`/chat`）
2. 观察左侧会话列表

**预期结果：**
- [ ] 会话列表不是扁平列表，而是**按智能体名称分组**的结构
- [ ] 每组以智能体名称为**分组标题**（如"hermes-coding"、"agent-coding"）
- [ ] 同一智能体的多个会话折叠在该分组下
- [ ] 分组标题旁有智能体图标标识
- [ ] 每组内的会话按**更新时间倒序**排列

---

### TC-02：会话列表点击导航

**前置条件：** 存在按智能体分组的会话列表

**测试步骤：**
1. 点击某个分组标题 → 展开/折叠该分组
2. 点击分组内的某个会话名称

**预期结果：**
- [ ] 点击分组标题可展开/折叠该组会话
- [ ] 点击会话名称 → 右侧显示该会话的聊天内容
- [ ] 当前选中会话高亮显示
- [ ] URL 更新为 `/chat/{sessionId}`

---

### TC-03：新建会话并关联智能体

**前置条件：** 已配置至少一个智能体

**测试步骤：**
1. 点击"新建会话"按钮
2. 在弹窗中选择智能体（如 hermes-coding）
3. 输入会话名称
4. 点击确认

**预期结果：**
- [ ] 新建的会话出现在对应智能体分组下
- [ ] 会话名称正确显示
- [ ] 自动跳转到该会话的聊天页面
- [ ] 后端正确记录了 `agentId` 关联

---

## 二、Header 显示智能体信息

### TC-04：收起侧栏时 Header 显示智能体信息

**前置条件：** 已打开某个会话

**测试步骤：**
1. 进入聊天页面，打开某个会话
2. 点击侧栏收起按钮（`<`），使侧栏折叠

**预期结果：**
- [ ] 聊天窗口顶部出现 **Header 区域**
- [ ] Header 第一行显示：**会话名称**
- [ ] Header 第二行显示：**🤖 + 智能体名称**
- [ ] Header 第二行右侧显示：**智能体类型标签**（Hermes=绿色，Goldfish=橙色，其他=蓝色）
- [ ] Header 在侧栏展开时**依然可见**

---

### TC-05：切换会话时 Header 同步更新

**前置条件：** 两个不同智能体的会话已创建

**测试步骤：**
1. 打开会话A（属于智能体X）→ 观察 Header
2. 切换到会话B（属于智能体Y）→ 观察 Header

**预期结果：**
- [ ] 切换到新会话后，Header 的会话名称、智能体名称、类型标签**同步更新**
- [ ] 无刷新闪烁或残留旧数据

---

## 三、真实对话服务连接

### TC-06：Hermes 类型智能体真实对话

**前置条件：** 已配置一个 Hermes 类型的智能体（baseUrl、apiKey、agentId 正确）

**测试步骤：**
1. 创建或选择一个关联该 Hermes 智能体的会话
2. 在输入框中输入测试消息（如"你好"）
3. 点击发送

**预期结果：**
- [ ] 发送成功后，用户消息立即显示在聊天区域
- [ ] **不出现**"[Hermes 适配器错误]"等错误提示
- [ ] 后端正确通过 Hermes API Server 转发请求
- [ ] 智能体的回复内容正确返回并显示
- [ ] 消息**保存到数据库**（刷新页面后依然存在）
- [ ] 支持**流式输出**（文字逐字显示效果）

---

### TC-07：Hermes 适配器配置路由验证

**测试步骤：**
1. 打开浏览器 DevTools → Network 面板
2. 发送一条消息
3. 查看请求详情

**预期结果：**
- [ ] 请求发送到后端 `/api/chat/stream`
- [ ] 请求 body 包含 `agentId` 字段
- [ ] 后端日志显示选择了正确的适配器（HermesAdapter）
- [ ] HermesAdapter 正确读取 `HermesBaseUrl` 和 `HermesApiKey` 并转发到配置的 API Server
- [ ] SSE 流式响应正确接收 `data: {...}` 格式的数据

---

### TC-08：对话历史保持上下文

**前置条件：** 已完成 TC-06 的至少 3 轮对话

**测试步骤：**
1. 刷新页面
2. 重新进入同一个会话

**预期结果：**
- [ ] 之前的对话历史完整显示
- [ ] 继续发送新消息时，智能体能**记住上下文**（说明 Hermes API Server 正确处理了历史记录）

---

### TC-09：错误场景处理

**测试步骤：**
1. 临时修改一个智能体的 baseUrl 为无效地址
2. 尝试与该智能体对话

**预期结果：**
- [ ] 显示友好的错误提示（如"服务暂时不可用，请检查配置"）
- [ ] 不出现白屏或 JS 崩溃
- [ ] 错误信息记录到后端日志

---

## 四、综合场景

### TC-10：完整用户流程

**测试步骤：**
1. 打开 https://agent.ai.impx.net
2. 点击"聊天"导航
3. 点击"新建会话" → 选择 hermes-coding 智能体 → 创建会话
4. 在 Header 中确认显示了智能体名称和类型标签
5. 收起侧栏，确认 Header 信息仍然可见
6. 发送消息 → 验证真实对话
7. 再创建一个 agent-coding 的会话 → 验证会话列表分组正确
8. 切换到 agent-coding 会话 → 验证 Header 和分组都正确更新

**预期结果：** 以上所有步骤均无异常，数据正确显示
