## 1. 项目概述

### 1.1 项目背景

AgentFree（代号 Goldfish）是一个自主可控的智能体框架 MVP，旨在整合 Hermes 和 OpenClaw 框架的特性，构建一个简单但强壮的智能体运行平台。

### 1.2 项目目标

- 提供一个标准化的智能体运行环境
- 支持多种标准协议（A2UI、AG-UI、MCP、Skills、A2A）
- 内置常用工具集（文件操作、HTTP 请求、终端执行等）
- 提供简洁的管理后台

### 1.3 项目范围

**包含：**
- 智能体管理（CRUD）
- 协议适配器框架（AG-UI、A2UI）
- 聊天界面（支持 AG-UI、A2UI 协议，含会话管理）
- 内置工具集
- 管理后台（React + Vite）
- RESTful API（.NET 10）

**不包含（v1）：**
- 认证授权系统
- 分布式部署
- 消息队列
- 监控告警

## 2. 功能需求

### 2.1 智能体管理

#### 2.1.1 Agent 列表
- 显示所有 Agent 的基本信息
- 支持搜索和过滤
- 显示 Agent 状态（运行中/已停止/错误）

#### 2.1.2 Agent 创建
- 创建新的 Agent 实例
- 配置 Agent 名称、描述、协议类型
- 选择启用的工具集

#### 2.1.3 Agent 编辑
- 修改 Agent 配置
- 启用/禁用特定工具
- 修改协议适配器

#### 2.1.4 Agent 详情
- 查看 Agent 完整信息
- 查看执行日志
- 查看运行状态

#### 2.1.5 Agent 删除
- 删除 Agent（软删除）
- 确认对话框

### 2.2 协议适配器

#### 2.2.1 适配器接口
定义统一的适配器接口，支持：
- 消息接收
- 消息发送
- 会话管理
- 工具调用

#### 2.2.2 内置适配器
- MCP 协议适配器
- A2UI 协议适配器
- AG-UI 协议适配器（预留）
- Skills 协议适配器（预留）
- A2A 协议适配器（预留）

### 2.3 内置工具

#### 2.3.1 文件工具
- 读取文件内容
- 写入文件内容
- 列出目录
- 文件复制/移动

#### 2.3.2 HTTP 工具
- GET 请求
- POST 请求
- 自定义 Header
- 响应解析

#### 2.3.3 终端工具
- 执行命令
- 获取输出
- 超时控制

#### 2.3.4 工具配置
- 通过配置文件动态加载
- 启用/禁用特定工具

### 2.4 管理后台

#### 2.4.1 页面结构
- 首页（Agent 概览）
- Agent 列表页
- Agent 创建/编辑页
- Agent 详情页

#### 2.4.2 技术要求
- React 18 + TypeScript
- Vite 构建工具
- 响应式布局
- 简洁的 UI 设计

## 3. 非功能需求

### 3.1 性能
- API 响应时间 < 200ms（简单查询）
- 支持并发 Agent 执行

### 3.2 可靠性
- 服务可用率 > 99%
- 异常处理完整，不吞异常

### 3.3 可维护性
- 代码符合项目编码规范
- 注释充分，关键逻辑有注释
- 日志记录关键操作和异常

### 3.4 安全性
- 输入验证
- 防止 SQL 注入
- 防止 XSS 攻击

### 4.3 数据库设计

#### Agent 表

| 字段 | 类型 | 说明 |
|------|------|------|
| Id | int | 主键，自增 |
| Name | varchar(100) | Agent 名称 |
| Description | text | 描述 |
| ProtocolType | varchar(50) | 协议类型 |
| Status | varchar(20) | 状态（Active/Inactive/Error） |
| CreatedAt | datetime | 创建时间 |
| UpdatedAt | datetime | 更新时间 |

## 5. API 接口设计

### 5.1 Agent 管理 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/agents | 获取 Agent 列表 |
| GET | /api/agents/{id} | 获取 Agent 详情 |
| POST | /api/agents | 创建 Agent |
| PUT | /api/agents/{id} | 更新 Agent |
| DELETE | /api/agents/{id} | 删除 Agent |
| POST | /api/agents/{id}/start | 启动 Agent |
| POST | /api/agents/{id}/stop | 停止 Agent |

### 5.2 协议适配器 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/adapters | 获取适配器列表 |
| POST | /api/adapters/register | 注册适配器 |

### 5.3 工具 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/tools | 获取可用工具列表 |
| POST | /api/tools/{name}/execute | 执行工具 |

## 7. 风险与约束

### 7.1 风险
- 标准协议规范尚未最终确定
- 多协议适配器的复杂度可能超出预期

### 7.2 约束
- v1 不包含认证授权
- 单线程 Agent 执行
- 数据库复用现有 PostgreSQL 实例
