# AgentFree 需求规格说明书

> 版本: 1.0 | 日期: 2026-05-21 | 状态: 🟢 需求确认中

---

## 一、项目定位

AgentFree 是一个自主可控的智能体框架，目标是：

- **整合现有框架优势**：吸收 Hermes 的 SOUL.md 驱动能力和 OpenClaw 的通道管理、会话管理、技能系统能力
- **支持标准协议**：统一对接业界标准协议，降低智能体间的耦合和互操作性成本
- **提供可视化平台**：通过 Web 控制台管理智能体生命周期、监控运行状态、管理技能

---

## 二、核心能力需求

### 2.1 智能体运行时（Agent Runtime）

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| REQ-001 | 智能体注册与管理 | P0 | 支持注册/注销/启停智能体实例 |
| REQ-002 | SOUL.md 驱动 | P0 | 每个智能体通过 SOUL.md 定义身份、职责、红线 |
| REQ-003 | 会话管理 | P0 | 会话创建、历史管理、压缩（compaction）、持久化 |
| REQ-004 | 多模型路由 | P0 | 支持多 provider 模型配置，主模型 + fallback |
| REQ-005 | 工具调用框架 | P0 | 统一的工具执行引擎，支持工具注册、权限控制 |
| REQ-006 | 通道适配层 | P1 | 支持企业微信、飞书等消息通道 |
| REQ-007 | 技能系统 | P1 | 技能发现、加载、执行（类似 OpenClaw Skills 机制） |

### 2.2 标准协议支持

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| REQ-100 | MCP Server | P0 | 支持作为 MCP Server，暴露数据和工具 |
| REQ-101 | MCP Client | P1 | 支持作为 MCP Client 连接外部 MCP Server |
| REQ-102 | A2UI 渲染 | P1 | 支持 A2UI 协议的 UI 生成和渲染对接 |
| REQ-103 | AG-UI 流式交互 | P1 | 支持 AG-UI 协议的流式对话交互 |
| REQ-104 | A2A 协议 | P2 | 支持 Agent-to-Agent 跨智能体通信 |
| REQ-105 | Skills 标准化 | P1 | 技能定义、注册、发现的标准化 |

### 2.3 智能体协作

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| REQ-200 | 子智能体生成 | P0 | 支持主智能体动态生成子智能体（subagent） |
| REQ-201 | 任务委派 | P0 | 支持任务分解、委派、结果回收 |
| REQ-202 | 智能体间消息 | P1 | 支持跨智能体的消息传递 |
| REQ-203 | 会话隔离 | P0 | 每个智能体有独立的会话空间 |

### 2.4 记忆系统

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| REQ-300 | 短期记忆（上下文） | P0 | 会话上下文管理，支持压缩策略 |
| REQ-301 | 长期记忆（文件） | P0 | 基于文件的记忆系统（MEMORY.md + memory/*.md） |
| REQ-302 | 语义检索 | P1 | 支持向量化的语义搜索能力 |

### 2.5 Web 控制台

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| REQ-400 | 智能体管理界面 | P0 | 创建、编辑、启停智能体 |
| REQ-401 | 会话管理界面 | P0 | 查看会话列表、历史、上下文 |
| REQ-402 | 技能管理界面 | P1 | 查看和注册技能 |
| REQ-403 | 配置管理 | P0 | 模型配置、通道配置、路由配置 |
| REQ-404 | 监控面板 | P1 | 运行状态、API 调用统计、费用监控 |

## 三、非功能性需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NFR-001 | 部署方式 | 本地部署优先（macOS），后续支持 Docker/K8s |
| NFR-002 | 数据安全 | 密钥凭据加密存储，不暴露到日志 |
| NFR-003 | 可扩展性 | 插件化架构，新协议/新通道可热插拔 |
| NFR-004 | 性能 | API 响应 < 200ms（不含 LLM 调用），支持并发 |
| NFR-005 | 安全性 | 权限控制、沙盒隔离、红线强制 |

## 四、与现有系统关系

```
AgentFree (本项目)
├── 运行时核心
│   ├── 整合 Hermes: SOUL.md 驱动、实例管理
│   └── 整合 OpenClaw: 通道管理、会话管理、技能系统
├── 标准协议层
│   ├── MCP Server/Client
│   ├── A2UI 支持
│   ├── AG-UI 支持
│   └── A2A 支持
└── Web 控制台
    └── 智能体管理 + 监控

LLM-FREE (独立项目 P001)
└── 网关平台（不在此项目范围内，但可借鉴其架构经验）
```
